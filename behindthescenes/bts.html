💻 Behind the Scenes

I made this webpage for those who are interested in knowing how this website was made. But that's not the only reason for this: i made this website also for those who may want to know how to build their own website and deploy it.
Note: this webpage could have bad content. The structure of the page, the text, or the images could be changed at any moment.

The most important tools I've used

You're probably asking yourself "What did this guy use to make his own website?". In this section of the website I'll tell you the most important tools I've used to make all this. With "tools" I mean programs and websites.

Visual Studio Code

In my opinion this is one of the best IDEs you can find. There is a reason if many people recommend it. Wanna take a look at the official website? Click here.

GitHub

GitHub is an awesome website for developers. I'm currently using it to make this website open-source, and to allow people (and me) to report issues for this website. The most important thing for me is that GitHub has a feature called GitHub Pages, which allows you to deploy your website on their servers for free. They'll give you a free link, but if you want you can buy a domain and use it on your website. Click here to go to the GitHub's website.

Live Server

Live Server is an extension for VS Code. It allows you to create a web server directly on your computer to preview the website you're currently writing. It also has live reload. That means that to update the website preview you don't need to stop the server and restart it, you can simply save the file you're currently editing. That's a really fast thing to do, just press Ctrl + S or CMD + S.

Codecademy

Codecademy is an awesome platform to learn how to code. Because of it now I'm able to build static websites from scratch. But that's not all. You can learn other languages such as JavaScript, Python, C++ and many more. This is not sponsored, it's just that I love this website and I think other people will find it useful as much as I do. Click here to access Codecademy's website.

The starting of the website

Before starting all the coding stuff I made a wireframe of the website. A wireframe is just an image representing only the structure of the website. For the text I used the classic lorem ipsum text, since this was just for me. To make this wireframe I used Vectornator, a vector design program for iPhone, iPad and Mac. I used the iPad version to make this wireframe. Here's an image of it. You'll also notice that the design is different from the actual website. That's because I had numerous ideas when developing the website, but after trying all of them I decided that the actual design was the best one!

After everything the 1st version of the website was made. But (of course) it wasn't ready yet. There were lots of things missing, the most important ones being dark mode and website responsiveness. But what is website responsiveness?
The responsiveness of a website is its ability to resize and change the design for the various screen resolutions available out there. In the PC world it could be easier to make a website responsive, because monitors have common resolutions like 1280x720, 1920x1080, 2560x1440 and more. But if we think about mobile devices, do they all have the same screen size? No. Some phones are bigger, some are smaller, some have a notch, some have a hole-punch camera, and some are all-screen. We have to make our websites easily usable to everyone, regardless of them using a smartphone, computer or tablet.
I wanted to add dark mode too because a lot of people use it, and in my opinion it looks cooler than light mode. You don't want to end like this guy, right?

While doing all this I tried to give the website a modern design by using readable fonts and rounded corners, but I also tried to make website navigation as easy as possible. This is something I personally think all website developers should do.

What about colors?

Colors are a fundamental part of every website. Without them websites would be plain boring. Colors can be defined with various types of values, but the ones I used were the hex ones and the rgba ones. There are also other types of values such as hue and hsl. To help myself with all the color codes and with the various shades I used the W3 Schools Color Picker. Pick a color and you'll see previews of black and white text, both with and without shadows, lighter/darker colors, hue, saturation and lightness. And (of course) you'll have the hex, rgb, hue and hsl values. (Not sponsored again, I just like the website and I think it'll be useful for other people too).

Developers also use colors to define how important or not is something. For example see how the titles are completely black and the paragraphs are dark grey. This is something I did to make the two more distinguishable from each other. And in dark mode you'll see that the titles are white and the paragraphs are light grey. You can also see how the "BETA" text in the navbar is inside a red box. That red box draws attention, so everyone will see that this website isn't fully ready yet.

How do you remember all the various things in website developing?

Most people think that developers are people with superpowers that make them remember every single thing. But that's not true. The truth is that they don't. Instead we remember what we can, and if we don't remember something, we have a problem to solve. That's what programmers do the most: resolve problems. When we have a problem we try to solve it with what we remember, but if we don't, Google comes to help us. A quick research can make us remember lots of things. And here I'll mention another tool that a lot of developers use: MDN Web Docs. These are a lifesaver when you don't remember something. (not sponsored again)

How much time do you need to make a website?

Developing a website can take as little as a few hours, but also as much as a month, or even more. It depends entirely on your skills and the complexity of the website you're making. The same is for making new features, it can take just a few hours, an entire day or even weeks. To make this webpage it took me around 3/4 hours of coding. That's excluding the things I deleted while making the webpage and the time it took to think about what to write here and what not.

I had fun while making this website, and I'm looking forward to improve it in every single aspect!
